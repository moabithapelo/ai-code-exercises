After decomposition:

2. What was the most challenging part of decomposing the function?

The most challenging part was handling shared state and dependencies.

In the original function:

    sales_data was modified multiple times (date filtering, then filter application)

    grouped_data was used in multiple places (report building, chart generation)

    total_sales was needed for percentage calculations in grouping

How I addressed it:

    Passing data explicitly: Each function receives the data it needs and returns transformed data

    Clear data flow: The main function shows exactly how data moves through the pipeline

    No side effects: Helper functions don't modify input data; they return new data

    Returning structured results: calculate_basic_metrics returns a dictionary, not individual values

Example challenge: The original function calculated total_sales early, then used it later for grouping percentages. In the refactored version, I had to:

    Return metrics as a structured object

    Pass metrics['total_sales'] to add_grouping_to_report

    Ensure grouping function didn't need to recalculate totals
